The Open Name System

The Open Name System (or ONS) is a protocol that builds on top of the domain name system and provides the additional functionality of the resolution of usernames to user identities.

User identities are represented as JSON data, and username-to-user-data mappings are stored in a decentralized key-value store.

Any KV store may have multiple namespaces, and currently the Namecoin blockchain is home to the largest namespace in existence.

opendig

"opendig" is a command line tool that extends the functionality of dig to support the resolution of usernames. opendig supports all dig calls, as well as username-specific calls.

opendig user data
$ opendig -u justin

{
    "name": {
        "formatted": "Justin Kan"
    },
    "website": "http://justinkan.com",
    "bitcoin": {
        "address": "1Jf5u4LiYuN3S5Fhj1At7fooSbPhQSMk8j"
    },
    ...
    }
}
opendig bitcoin data
$ opendig bitcoin -u justin

{
    "address": "1Jf5u4LiYuN3S5Fhj1At7fooSbPhQSMk8j"
}

user schema

ONS and opendig currently use the v0.2 user-schema, but a new standard (v0.3) is being developed, which is in the RFC phase. After sufficient people have reviewed, commented on, and revised the v0.3 schema, a rollout of the new version will begin.

v0.2
{
"name": {
    "formatted": "Ryan Shea"
},
"bitcoin": {
    "address": "14eautXfJT7EZsKfm1eHSAPnHkn3w1XF9R"
},
"twitter": {
    "username": "ryaneshea",
    "proof": {
        "url": "https://twitter.com/ryaneshea/status/486057647808868352"
    }
},
"v": "0.2",
}
v0.3
{
"basics": {
    "name": "Ryan Shea",
    "location": "New York, NY",
},
"payments": [
    { "type": "bitcoin", "address": "14eautXfJT7EZsKfm1eHSAPnHkn3w1XF9R" },
],
"profiles": [
    { "type": "twitter", "username": "ryaneshea",
      "proof": "https://twitter.com/ryaneshea/status/486057647808868352" },
],
"v": "0.3",
}
